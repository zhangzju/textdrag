ALG
ALG (Gateway de Camada de Aplicação) permite filtros tranversais NAT a serem plugados no gateway a fim de serem compatíveis com tradução de endereço e porta para certos protocolos de camadas de aplicação \"control/data\": FTP, TFTP, H323 e etc. É recomendável habilitar ALG.
Passagem PPTP
Se ativado, ele permite que as sessões ponto-a-ponto sejam tuneladas através de uma rede IP e passadas pelo roteador.
Passagem L2TP
Se selecionada, permite sessões ponto-a- ponto L2 a serem tuneladas através de uma rede IP e passando pelo roteador.
Passagem IPSec
Se selecionada, permite IPSec (Internet Protocol Security) a ser tunelado através de uma rede IP passando pelo roteador. IPSec usa serviços de segurança criptográficos para garantir comunicações privadas e seguras através de redes IP.
ALG FTP
Se selecionada, permite clientes FTP (File Transfer Protocol) e servidores transferirem dados via NAT.
ALG TFTP
Se selecionada, permite clientes TFTP ( Trivial File Transfer Protocol ) e servidores transferirem dados através de NAT.
ALG H323
Se selecionado, ele permite que os clientes Microsoft NetMeeting possam se comunicar via NAT.
ALG SIP
Caso selecionada, permite que clientes se comuniquem com servidores SIP (Session Initiation Protocol) via NAT.

DMZ
A função host DMZ (Zona Desmilitarizada) permite um host local a ser exposto à Internet para propósitos específicos como jogos online ou vídeo-conferência. Basicamente, o DMZ permite que um único computador na LAN abra todos as suas portas. Este computador tem de ser configurado com um endereço IP estático e está com a sua função de cliente DHCP desativada. Quando DMZ está habilitado, o host DMZ está totalmente exposto à Internet, o que pode trazer algum risco de segurança potencial para o computador . Caso o DMZ não esteja em uso, favor desabilitá-lo em tempo.
passo
Para atribuir um computador ou servidor para serem um servidor DMZ
Selecione Habilitar DMZ.
No campo Endereço IP Host DMZ, digite o endereço IP de um computador local para configurar como o host DMZ.

Intervalo Beacon
Insira um valor entre 25 e 1000 em milissegundos para determinar a duração entre pacotes beacon que são transmitidos pelo roteador para sincronizar a rede wireless. O padrão é 100 milisegundos.
RTS Threshold
Insira um valor entre 1 e 2347 para determinar o tamanho do pacote de transmissão de dados através do roteador. Por padrão, o tamanho Threshold RTS (Request to Send) é de 2.347. Se o tamanho do pacote é maior do que o limite predeterminado, o roteador envia um pedido de envio de quadros a uma estação receptora e negocia o envio de uma frame de dados, ou então o pacote será enviado imediatamente.
Intervalo DTIM
Este valor determina o interno de DTIM (Mensagem de Indicação de Tráfego de Entrega). Insira um valor entre 1 e 255 internos. O intervalo padrão é 1, indicando o intervalo DTIM é o mesmo que o Intervalo Beacon.
Período de Atualização de Chave de Grupo
Digite o número de segundos para controlar o intervalo de tempo para a renovação automática da chave de criptografia. O padrão é 0, indicando que não há renovação de chave.

A função WMM garante os pacotes de mensagens com alta prioridade a serem transmitidos preferencialmente. O WMM está habilitado compulsivamente no modo 802.11n ou 802.11ac. É altamente recomendável habilitar WMM.

Modo Operacional

Neste modo, o roteador se conecta diretamente à Internet via IP dinâmico, IP estático, PPPoE, L2TP ou PPTP e compartilha o acesso à internet a vários dispositivos com ou sem fio. Access Point
Neste modo, seu roteador se conecta a um roteador com ou sem fio através de um cabo Ethernet e amplia a cobertura wireless da sua rede existente. Funções como NAT, Controle dos Pais e QoS não são suportadas neste modo. O endereço IP deste roteador é atribuído pelo servidor DHCP do roteador raíz. Se você não conhece o endereço IP deste roteador, pode usar http://tplinkwifi.net para fazer login na página de gerenciamento da web.

DSCP para SIP e para RTP
DSCP (Differentiated Services Code Point) são os primeiros 6 bits no byte ToS (Type of Service). A marcação DSCP permite garantir um tratamento preferencial para o tráfego de maior prioridade na rede com base no valor DSCP. Selecione DSCP para SIP (Session Initiation Protocol) e RTP (Real-time Transport Protocol), respectivamente. Se você não tiver certeza, mantenha o valor padrão.
Configurações de Relay DTMF
 Selecione um dos seguintes protocolos para a configuração do relé DTMF. Se você não tem certeza sobre qual deles selecionar, mantenha o valor padrão.
 INFO SIP
Selecione esta opção para enviar tons DTMF na mensagem SIP INFO para o fim remoto.
RFC2833
Selecione esta opção para capturar o número inserido no teclado do telefone e, em seguida, convertê-lo em uma forma digital e enviá-lo para o outro extremo; o receptor irá gerar o tom de acordo com a forma digital que recebe. Esta opção é recomendada, pois ainda pode ser a precisão do tom DTMF quando ocorre o congestionamento do tráfego da rede.
In-Band
Selecione esta opção para enviar o tom DTMF como um áudio quando tocar no teclado no telefone.
